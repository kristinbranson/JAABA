Visualizing Classifier

JAABA uses boosting learning algorithm to train classifiers that are used to detect behaviors. Boosting works by combining simple rules that are typically not very accurate into an accurate rule. In JAABA, we use decision stumps as the simple rules. Decision stumps predict by simply comparing a window features value to a threshold. With the large number of window features we compute, the simple decision stumps are quite effective in learning an accurate detector.

As the weak rules are straightforward, it is possible to visualize the classifier by looking at the window features that are used by the classifier. When you use Classifier -> Visualize Classifier, it tells the window features used by the classifier, threshold used on those feature, and the weight placed on feature by the current classifier.

Below is an example output. The features are ordered in the descending order of the weight that was placed on it. The feature names are listed on the right. The parts of the feature names are (in order) per-frame feature name, the window feature computation type, transformation type, radius of the window, and offset of the window. The leftmost figure shows that the weight that is given to the feature by the boosting algorithm. The middle part shows where the threshold (as a percentile of the feature value) is selected by the boosting algorithm for that features.
