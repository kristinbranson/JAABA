Installation and Starting JAABA


JAABA can either be run

  • The Easy Way using executables that can be downloaded and clicked on (most users) or
  • The Advanced Way from the MATLAB command line using our source code (advanced users, users with MATLAB licenses using the development version of the code).
Please follow the appropriate instructions below.

Minimum Computer Requirements

JAABA can run on Linux (64-bit), Windows (32- or 64-bit), or Mac OS X (64-bit) systems with at least 4GB RAM. JAABA uses parallel computation, and will be faster on multicore machines. JAABA is a computationally intensive program, and will run better on better machines. The amount of memory required scales with the complexity of the behavior.

Installation and Start-up: Easy

Most users should use the executable version of JAABA. This only requires downloading a program and clicking it.

Software Requirements (Easy)

The only requirement is the freely available Matlab Compiler Runtime (MCR) for R2012a. MCR installers and instructions are available at Mathworks website.

Installation (Easy)

  1. Download and install the Matlab Compiler Runtime (MCR) for R2012a. MCR installers and instructions are available at Mathworks website.
  2. Download the JAABA package for your system from http://sourceforge.net/projects/jaaba/files.
  3. Unzip the package zip file.
  4. Leave the executable in the directory in the package. You can move the entire directory, but JAABA relies on other files in that directory, so please leave them in the same directory.

Start-Up (Easy)

Within the package, there is an executable program. The name of this file depends on the OS:
  • Windows: Double-click JAABA.exe to run JAABA. The first time you run this program, you may receive a Windows Security Warning from your firewall. This is normal, and we recommend allowing JAABA to run. See Windows Firewall Security Warning below for more information.
  • Mac OS X: Double-click the JAABA application to run JAABA.
  • Linux: The shell script run_JAABA.sh runs JAABA. It should be run from within the JAABA package directory using the command 
    ./run_JAABA.sh [MCR location]
Please be patient. Loading the MATLAB MCR is slow, hence once you run the executable it may take some time for JAABA to appear.

Installation and Start-up: Advanced

Users with programming experience, with MATLAB licenses, and/or wanting to use the development version of JAABA can run JAABA from the source code within MATLAB.

Software Requirements (Advanced)

  • MATLAB version >= 2012a (7.14.0.739)
  • MATLAB Parallel Computing, Image Processing, and Statistics Toolboxes

Installation (Advanced)

  1. Download the source code. There are two options:
    • github: Clone the JAABA git repository at github.
    • zip file: Download a source code zip file from Sourceforge.
  2. Change into the directory perframe: 
    cd [JAABA location]/perframe
  3. Run StartJAABA on the command line to start JAABA.

Windows Firewall Security Warning

When JAABA is started for the first time in Windows, it may trigger a Windows Security Warning from your firewall. This is normal, and we recommend you allow JAABA to proceed. This warning is triggered when JAABA calls matlabpool, part of MATLAB's Parallel Computing Toolbox. The Parallel Computing Toolbox can be used to distribute jobs over a many computers over a network, which is why Windows creates a warning. JAABA only uses multiprocessing on a single machine, and thus will not be accessing your network.
