Labels and Classifiers

Saving Labels

The current set of manual labels for all videos can be saved by selecting the menu item File -> Save Labels. The labels for each video get saved in a file in its corresponding experiment directory. The name of this file is the same for all experiments, and is specified in the configuration file (value of the file.labelfilename parameter). We will refer to this labels file in each experiment directory as the default location. The file.labelfilename parameter is set to labeled[ProjectName].mat by default.

Loading Labels

If you close JAABA, start a new JAABA session, open this project, and add this experiment, these saved labels are automatically loaded. The label filename should be different for each different project, so if you add the same experiment, but select a different project, labels for the current project won't be loaded.

Saving the Classifier and Labels

Similarly, File -> Save classifier and labels will save both the current classifier and the labels. The labels are saved to their default location. JAABA will prompt you each time for the location to save the classifier to.

Loading a Classifier

A saved classifier can be loaded using the Load and Load w/o exps buttons in the Edit Files dialog when starting a new JAABA session.

  • Load will load a selected classifier along with all the videos/experiments that were used to train that classifier while
  • Load w/o exps loads the classifier without any of the experiments.
These same choices are also available under the File -> Load Classifier menu.

Classifier's Predictions

Saving Predictions

After training a classifier, the whole trajectory of a animal, the whole experiment, or all the experiments can be classified by selecting one of the options under the Classifier -> Classify menu, see Examining the Classifier's Predictions for details. The classifier's predictions and raw output scores can be saved to a file for several of the options under this menu.

  • Current animal: Predicts the behavior for all frames for the current animal and allows the user to view these results within the current JAABA session, but does not save the predictions to a file.
  • Classifier -> Classify -> Current Experiment -> Don't Save: Classifies all animals for the current experiment/video, and allows the user to view these results within the current JAABA session, but does not save the predictions to file.
  • Classifier -> Classify -> Current Experiment -> Save to default: Classifies all animals for the current experiment/video and saves to the scores file within the corresponding experiment directory. The name of the scores file within the experiment directory is the same for all experiments, and is specified in the project configuration parameter file.scorefilename.
  • Classifier -> Classify -> Current Experiment -> Save to different: Classifies all animals for the current experiment/video, then prompts the user for the location to save the scores to, and saves to this file.
    • Classifier -> Classify -> All Experiments -> Don't Save: Classifies all animals for all experiments/videos, and allows the user to view these results within the current JAABA session, but does not save the predictions to file.
  • Classifier -> Classify -> All Experiments -> Save to default: Classifies all animals for all experiments/videos, then saves to the scores file within each corresponding experiment directory. The name of the scores file within the experiment directory is the same for all experiments, and is specified in the project configuration parameter file.scorefilename.
  • Classifier -> Classify -> All Experiments -> Save to different: Classifies all animals for all experiments/videos, then prompts the user for a filename that will be used for all experiments. The classifier's predictions for each experment are stored into this selected filename within its corresponding experiment directory.

Loading Predictions

Previously saved predictions can be loaded from file into the current JAABA session. These loaded scores will be shown in the bottom portion of the Automatic Timeline if Loaded is selected from the pop-up menu on the left. Predictions can be loaded by selecting any of the options under the menu File -> Load Scores:

  • File -> Load Scores -> For current experiment from default location: Loads the scores for the current experiment from the default scores file within its corresponding experiment directory. The name of the scores file within the experiment directory is the same for all experiments, and is specified in the project configuration parameter file.scorefilename
  • File -> Load Scores -> For current experiment from different location: Prompts the user for a file location, and loads the scores for the current experiment from that file.
  • File -> Load Scores -> For all experiments from default location: Loads the scores for all experiments from the default scores file within each corresponding experiment directory. The name of the scores file within the experiment directory is the same for all experiments, and is specified in the project configuration parameter file.scorefilename
Options not mentioned above should not be used, and are there for backwards compatability purposes.
